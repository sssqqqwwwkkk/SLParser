# SLParser - StudentLibrary Book Parser 📚

Скрипт для браузерной консоли, позволяющий экспортировать полностью легально
доступную пользователю книгу с сайта StudentLibrary из режима «виртуального
скроллинга» в автономный, чистый и структурированный HTML-файл.

## 📌 Какую проблему решает скрипт?

Многие образовательные платформы используют технологию *Virtual Scrolling* —
текст рендерится только в видимой области экрана и скрывается при прокрутке.
Из-за этого:

- невозможно выделить и скопировать несколько страниц текста подряд;
- стандартное сохранение страницы даёт пустые листы или обрывки текста;
- неудобно читать при плохом интернет-соединении.

---

## 🚀 Что делает скрипт?

1. **Извлекает полный текст** — автоматически подгружает и склеивает все главы
   через API сайта на основе текущей авторизованной сессии.
2. **Генерирует титульный лист** — обложка, авторы, год, издательство, ISBN,
   аннотация.
3. **Восстанавливает структуру** — иерархия заголовков и кликабельное
   оглавление.
4. **Очищает стили** — убирает враждебные CSS-правила сайта и оформляет текст
   в классическом книжном стиле (красная строка, выравнивание по ширине).
5. **Встраивает изображения** — все картинки загружаются прямо во время
   активной сессии и сохраняются внутри файла в формате Base64, поэтому файл
   полностью автономен и не требует интернета при открытии.
6. **Автоматически скачивает результат** — по завершении парсинга браузер сам
   предложит сохранить готовый `.html`-файл.

---

## 📦 Формат результата

Скрипт сохраняет книгу как **единый автономный HTML-файл** — все тексты,
стили и изображения упакованы внутри одного документа.

Это промежуточный универсальный формат, который легко конвертировать в

## 🛠 Инструкция

1. Зайдите на сайт электронной библиотеки в свой аккаунт.
2. Откройте нужную книгу (к которой у вас есть легальный доступ).
3. Нажмите `F12`, перейдите во вкладку **Console (Консоль)**.
4. Скопируйте весь код из файла `SLParser.js` и вставьте в консоль.
5. Нажмите `Enter` и дождитесь завершения — жёлтый индикатор в правом верхнем
   углу будет показывать текущий прогресс.
6. Когда появится надпись **✅ Сборник успешно сформирован!**, браузер
   автоматически скачает готовый HTML-файл.
7. Откройте файл в любом удобном инструменте для дальнейшей конвертации.

## 👨‍💻 Технические детали

- Использует глобальные переменные страницы `reader_book` (ID книги) и
  `reader_access_token` (JWT-токен сессии) для нативных `fetch` POST-запросов
  к `/api/3.0/mb4rdr-bff/retrieve/*`.
- Между запросами к API встроена задержка **150 мс** для предотвращения
  блокировки (Rate Limit).
- Изображения загружаются с `credentials: include` и Bearer-токеном, после
  чего конвертируются в Base64 и встраиваются прямо в HTML — итоговый файл
  полностью автономен.
- Размер файла может быть заметно больше оригинала из-за Base64-кодирования
  изображений (+~33% к размеру каждой картинки).
